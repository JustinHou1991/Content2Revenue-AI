## 一、1-2 分钟项目精简汇报（面试口述版）

大家好，我介绍一下我的项目 **Content2Revenue AI**，这是一个 AI 驱动的内容-商业转化智能平台。

**项目背景：** 内容营销团队在抖音等平台大量产出脚本，但面临一个核心痛点——不知道哪些内容适合推给哪些潜在客户，缺乏数据驱动的匹配和策略建议。

**核心方案：** 我设计了一个四步 Pipeline：首先用大语言模型分析抖音脚本，提取 Hook 策略、情感曲线、CTA 设计等 15 个维度的特征；然后对销售线索构建 360° 用户画像，包含行业、痛点、购买阶段等维度并自动评分分级；接着通过五维度语义匹配引擎，计算内容与线索的契合度；最后基于匹配结果生成可执行的内容策略和 A/B 测试方案。

**技术架构：** 项目采用四层架构——UI 层用 Streamlit 构建 7 个功能页面，服务层包含 15 个核心模块，基础设施层涵盖认证、熔断、事件总线等 16 个模块，数据层用 SQLite 持久化。整个流程由 Orchestrator 编排器串联。

**设计模式：** 系统性地运用了 10 种设计模式。最核心的是模板方法模式——BaseAnalyzer 定义了标准的六步分析流程，子类只需实现 Prompt 构建和响应解析即可扩展新的分析能力。还有策略模式用于归因模型切换、工厂模式用于 LLM 客户端创建、编排器模式用于流程协调。

**技术亮点：** 几个我认为比较有价值的点——第一，LLM 调用层面实现了多模型适配，内置支持 DeepSeek、通义千问、商汤等 10 个模型，还支持运行时动态注册自定义模型；第二，针对 LLM 返回 JSON 不稳定的问题，实现了三级自动修复机制；第三，安全层面做了 Prompt 注入防护、API Key 加密存储、输入验证等多层防护；第四，内置了完整的 A/B 测试统计引擎，包括 Z 检验和效应量计算。

**反馈闭环：** 策略执行后可以回传反馈数据，系统会根据反馈梯度调整评分基准，形成持续优化的闭环。

整体代码量约 1.5 万行，有 20 多个测试用例，部署在 Streamlit Cloud 上。

## 三、面试高频问题 Q&A 备忘

### Q1：为什么选择这个技术栈？
- **Streamlit**：快速构建数据应用原型，适合 AI/ML 场景，开发效率高
- **SQLite**：轻量级、零配置、适合中小规模数据，WAL 模式支持并发读
- **OpenAI SDK**：统一接口适配多家 LLM 厂商，降低切换成本

### Q2：如何保证 LLM 返回结果的稳定性？
三层保障：(1) 系统级——Prompt 注入防护 + XML 标签隔离用户输入；(2) 解析级——JSON 三级自动修复（提取 → 清理 → LLM 自修复）；(3) 调用级——429/5xx 指数退避重试 + LRU 缓存减少重复调用。

### Q3：系统如何扩展新的分析能力？
通过模板方法模式，只需继承 BaseAnalyzer，实现 `_build_prompt()` 和 `_parse_response()` 两个方法即可。例如要增加"竞品分析"能力，创建 CompetitorAnalyzer 类，定义 Prompt 模板和输出字段映射即可，无需修改编排器代码。

### Q4：如何处理并发和安全？
- 并发：MatchEngine 使用 ThreadPoolExecutor 并发执行匹配任务，threading.Lock 保证线程安全
- 安全：JWT 认证、多租户隔离、API 限流、熔断器、输入验证（XSS/SQL注入/Prompt注入）、审计日志

### Q5：成本如何控制？
- LLM 缓存：SHA256 键 + LRU + TTL，避免重复调用
- 成本监控：实时追踪 Token 用量，按模型定价计算成本
- 优化建议：自动检测高成本模型和高频操作，推荐替代方案
- 退避重试：429 限流时指数退避，避免浪费 API 调用
