Project Overview
项目定位与核心价值
面向内容营销团队的一站式 AI 分析平台，解决「内容生产 → 线索匹配 → 策略优化」全链路效率问题。
📝

内容分析引擎

基于 LLM 的抖音脚本深度分析，提取 Hook 策略、情感曲线、叙事结构、CTA 设计等 15+ 维度特征，输出内容评分与改进建议。

👤

线索画像构建

从销售线索数据中构建 360° 用户画像，包含行业、痛点、购买阶段、决策因素、意向评分等维度，自动分级（A/B+/B/C/D）。

🎯

语义匹配引擎

内容与线索的五维度语义匹配（受众契合度、痛点相关性、阶段对齐度、CTA 适配度、情感共鸣度），支持批量并发匹配。

💡

AI 策略顾问

基于匹配结果生成可执行策略：内容策略、分发策略、转化预测、A/B 测试方案，形成「分析 → 匹配 → 策略 → 反馈」闭环。

📊

成本监控体系

实时追踪 Token 用量与 API 成本，按模型/时间维度统计分析，提供成本优化建议，支持数据导出。

🔒

安全与合规

Prompt 注入防护、API Key 加密存储（Fernet + XOR 降级）、输入验证、审计日志、JWT 认证、多租户隔离。

Core Modules
核心服务模块详解
15 个核心服务模块，覆盖分析、匹配、策略、缓存、安全等完整能力。
🧠

BaseAnalyzer — 分析器基类

模板方法模式定义标准 6 步分析流程：验证输入 → 构建 Prompt → 调用 LLM → 解析响应 → 校验输出 → 记录历史。内置 Prompt 注入防护（XML 标签包裹）、字段类型安全保证、批量分析（进度回调 + 取消事件）。

🔗

LLMClient — 统一 LLM 调用

支持 10+ 模型（DeepSeek/通义千问/商汤/LongCat/自定义），内置 429/5xx 指数退避重试、JSON 自动修复、流式输出、批量并发控制、Token 成本追踪。运行时动态注册自定义 OpenAI 兼容模型。

📦

Orchestrator — 流程编排器

端到端 Pipeline 编排，串联四大分析器。支持单步调用和完整流水线，批量操作使用单事务保存保证一致性。仪表盘数据聚合优化（单连接查询）。

💾

Database — 数据持久化

SQLite + WAL 模式，8 张核心表，JSON 字段自动解析。API Key 使用 Fernet 对称加密存储（不可用时 XOR 降级），支持自动迁移加密方式。

⚡

ContentScoringModel — 规则评分

基于历史数据的轻量级评分系统（无需 ML 训练），四维度加权评分：Hook(30%)、CTA(25%)、Structure(25%)、Emotion(20%)。支持反馈学习梯度调整评分基准。

🧪

ABTestEngine — A/B 测试框架

5 种预定义变体模板（对照组/激进型/教育型/社交证明型/提问引导型）。双比例 Z 检验、Cohen's h 效应量、Bonferroni 校正、样本量计算。

📈

ContentAttribution — 归因分析

5 种归因模型：首次触点、最后触点、线性、时间衰减（7 天半衰期）、U 型。客户旅程追踪、多模型对比分析、渠道排名波动性计算。

🛡️

安全基础设施

JWT 认证、多租户隔离、API 限流、熔断器、连接池、输入验证（XSS/SQL注入/Prompt注入检测）、审计日志、事件总线、依赖注入容器。

Design Patterns
设计模式应用
项目中系统性地运用了 10 种经典设计模式，确保代码的可维护性与扩展性。
01

模板方法模式 BaseAnalyzer.analyze()

定义分析的标准 6 步流程骨架，子类通过实现 _build_prompt() 和 _parse_response() 定制具体行为。同样应用于 UI 层的 BasePage → AnalysisPage → MatchPage 继承体系。

02

策略模式 AttributionModel / VariantTemplate

归因引擎根据枚举选择不同权重计算策略（首次/最后/线性/时间衰减/U型）。A/B 测试引擎的变体模板对应不同的 Prompt 修改策略。

03

工厂模式 Orchestrator / LLMClient

Orchestrator 作为工厂创建并组装四大分析器。LLMClient 根据模型名称从配置注册表中选择配置创建客户端，支持运行时动态扩展。

04

编排器/中介者模式 Orchestrator

作为中间人协调四大分析器和数据库之间的交互，屏蔽内部复杂性，对外提供简洁的 full_pipeline() 接口。

05

缓存模式 CacheManager + LLMCache + @cached

三层缓存体系：方法级装饰器缓存、LLM 调用级缓存（SHA256 键 + LRU + TTL）、配置级缓存。命中率实时可查。

06

观察者模式 EventBus + progress_callback

事件总线实现发布-订阅机制。批量分析支持进度回调和取消事件，实现分析过程的实时反馈。

07

注册表模式 MODEL_CONFIGS + HealthChecker

LLM 模型配置注册表支持静态定义 + 运行时动态注册。健康检查系统支持自定义检查项注册。

08

熔断器模式 CircuitBreaker

独立的熔断器实现，防止级联故障。结合 API 限流和连接池管理，保障系统在高负载下的稳定性。

Technical Highlights
核心技术亮点
从 LLM 调用优化到安全防护，每个细节都经过精心设计。

🤖 多模型支持 + 运行时扩展

统一接口适配 10+ LLM 模型，支持运行时动态注册任意 OpenAI 兼容 API。不支持 json_mode 的模型自动降级为 Prompt 引导 + JSON 提取。

  • 内置模型：DeepSeek / 通义千问 / 商汤日日新 / LongCat
  • 自定义模型：register_custom_model() 运行时注册
  • 自动降级：json_mode 不可用时切换为 prompt 引导模式

🔒 Prompt 注入防护体系

多层防护机制确保 LLM 调用安全，防止用户输入中的恶意指令干扰系统行为。

  • 正则检测：识别 "忽略指令"、"you are now" 等注入模式
  • XML 标签包裹：用 <user_content> 标签隔离用户输入
  • 系统 Prompt 约束：明确指示 LLM 忽略用户输入中的指令
  • 输入截断：max_length=5000 防止超长输入攻击

🔄 JSON 自动修复机制

LLM 返回的 JSON 经常格式不规范，系统实现了三级修复策略。

  • 第一级：从 Markdown 代码块中提取 JSON
  • 第二级：正则清理（去除注释、尾逗号等）
  • 第三级：调用 LLM 自身修复损坏的 JSON（_repair_json）

💰 成本监控与优化

实时追踪每一次 LLM 调用的 Token 用量和成本，提供多维度分析和优化建议。

  • 线程安全的 Token 计数器（threading.Lock）
  • 按模型定价自动计算成本
  • 按日/周/月/模型维度统计分析
  • 智能优化建议：高成本模型检测、高频操作检测

🔐 加密存储（Fernet + XOR 降级）

API Key 等敏感数据使用双层加密方案，确保安全性同时保证可用性。

  • 优先使用 cryptography.fernet 对称加密
  • 不可用时自动降级到 XOR 加密
  • FERNET: 前缀标识加密类型，解密时自动识别
  • _migrate_to_fernet() 自动迁移旧数据

📊 A/B 测试统计引擎

内置完整的统计检验工具，确保实验结论的科学性。

  • 双比例 Z 检验（pooled standard error）
  • Cohen's h 效应量计算
  • Bonferroni 多重比较校正
  • 样本量计算（Abramowitz-Stegun 近似）
Database Design
数据库架构
SQLite + WAL 模式，8 张核心表覆盖全业务流程，JSON 字段自动解析。

📦 content_analysis

内容分析结果。存储原始脚本、LLM 分析结果（Hook/CTA/评分等 15+ 维度）、使用模型。

👤 lead_analysis

线索画像数据。存储原始线索、画像结果（行业/痛点/意向评分/等级）、使用模型。

🎯 match_results

匹配结果。五维度评分（受众/痛点/阶段/CTA/情感）、内容与线索快照、关联 ID。

💡 strategy_advice

策略建议。四大模块（内容策略/分发策略/转化预测/A/B测试）、关联匹配与内容线索。

⚙️ app_settings

系统配置。Key-Value 结构，API Key 加密存储，支持自定义模型配置持久化。

📊 api_usage

API 使用记录。Token 用量、成本、操作类型、关联内容/线索 ID，支持成本分析。

🔄 strategy_feedback

策略反馈。是否采纳、实际转化率、备注，用于反馈学习闭环。

🧪 ab_tests

A/B 测试。变体配置、测试结果、对照组标识，支持统计显著性检验。
